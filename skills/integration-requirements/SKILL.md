# 工作流 (Workflow)

1. 读取 RR、澄清、流程、单据字段和竞品参考。
2. 识别外部系统、系统责任、交互方向、触发时机和业务目的。
3. 梳理数据交换对象、业务事件、时效、失败处理、补偿和对账要求。
4. 输出集成需求和外部系统矩阵。
5. 回读产物，确认没有臆造技术 API、消息协议或字段。

# Tool Usage Notes

- 保持 BA 需求视角，只描述业务边界、交换内容和约束。
- 技术协议未知时写“待 IT 设计确认”，不要输出 OpenAPI/AsyncAPI。
- 集成场景必须追踪到 RR、流程或单据字段。

# 最终生成策略 (Final Generation)

1. 集成需求要覆盖参与系统、业务责任、触发时机、成功/失败处理和待确认项。
2. 系统矩阵 YAML 要包含系统、角色、交互方向、数据对象、时效、责任边界和确认状态。
3. 数据交换事件要说明事件、条件、数据内容、消费方、对账和补偿。
