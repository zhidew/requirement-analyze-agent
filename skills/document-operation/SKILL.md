# 工作流 (Workflow)

1. 读取 RR、澄清、规则和竞品材料，识别单据对象和作业场景。
2. 梳理字段、来源、必填性、展示/编辑条件、校验和默认值。
3. 梳理用户动作、状态变化、前置条件、后置结果和异常处理。
4. 输出角色、权限、动作和数据范围要求。
5. 回读产物，确认字段、动作、状态和规则引用一致。

# Tool Usage Notes

- 字段和动作必须有来源或明确标为建议/待确认。
- 不写技术 UI 实现、数据库 DDL 或接口契约。
- 对竞品能力只提炼需求启发，不直接照搬无关功能。

# 最终生成策略 (Final Generation)

1. 单据作业说明要覆盖对象、状态、动作、前后置条件、校验和异常。
2. 字段 YAML 要结构化输出字段名、含义、类型倾向、必填性、来源、校验、展示/编辑条件和追踪来源。
3. 权限说明要覆盖角色、动作、数据范围、敏感操作和待确认项。
